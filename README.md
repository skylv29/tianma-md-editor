# 天马 MD 编辑器

> 专为中文公文写作设计的 Markdown 桌面编辑器。  
> 基于 Electron + React + CodeMirror 6 构建，支持所见即所得编辑、实时预览，并一键导出符合国家标准的 Word 公文。

---

## 这个软件是为了解决什么问题而生的？

如果你用过 ChatGPT、Claude、文心一言等 AI 工具起草公文、报告或方案，一定遇到过这个让人抓狂的问题：

**AI 生成的内容，直接粘贴到 Word 里，格式全乱了。**

AI 的输出通常是 Markdown 格式——标题用 `#` 号表示，加粗用 `**` 包裹，列表用 `-` 开头。这套格式在 AI 对话界面里看起来整整齐齐，但一旦粘贴进 Word，会变成这样：

- `## 一、工作概述` 不会变成黑体大标题，而是原样出现一串井号
- `**重点内容**` 不会加粗，而是前后各带两个星号
- 列表缩进、段落间距全部乱掉
- 字体更是毫无规律，仿宋、宋体、微软雅黑混杂

于是你不得不在 Word 里手动一条条重新设置字体、字号、缩进，一份稍微长一点的公文，光整理格式就要花掉半小时甚至更久。这显然不是 AI 应该带来的体验。

**天马 MD 编辑器就是为了解决这个问题而生的。**

它的工作流程很简单：

1. 把 AI 生成的 Markdown 内容粘贴进来（或者直接在编辑器里用 Markdown 写作）
2. 实时预览公文排版效果，按需调整
3. 点击「导出公文 Word」，一键生成格式完全符合中国国家公文标准的 `.docx` 文件

导出的 Word 文件开箱即用：一级标题是方正小标宋、二级标题是黑体、三级标题是楷体、正文是仿宋，字号、行距、页边距全部按国标设置，不需要再动任何格式。

## 功能特性

### 三种编辑模式，按需切换

**所见即所得（WYSIWYG）模式**

日常写作的主要模式。Markdown 标记符号在光标离开后自动隐藏，标题自动切换为对应的公文字体和大小，`**加粗**`、`*斜体*`、行内代码等效果实时呈现，提供接近 Word 的沉浸式文档感，但底层仍然是干净的 Markdown 文本。

**分栏预览模式**

左侧显示原始 Markdown 源码，右侧实时渲染公文预览效果。适合在检查排版细节时使用，或者向他人展示最终效果时使用。

**源码模式**

等宽字体 + 行号显示，回归纯粹的文本编辑器体验。适合编写复杂表格、代码块，或需要精确控制 Markdown 格式时使用。

### 严格遵循国家公文字体规范

预览界面与导出的 Word 文件使用完全一致的字体方案：

| 元素 | 字体 | 大小 |
|---|---|---|
| 一级标题（h1） | 方正小标宋简体，居中 | 29px / 22磅 |
| 二级标题（h2） | 黑体 | 21px / 16磅 |
| 三级标题（h3） | 楷体\_GB2312 | 21px / 16磅 |
| 四级标题（h4） | 仿宋加粗 | 21px / 16磅 |
| 正文段落 | 仿宋\_GB2312，首行缩进 2 字符 | 21px / 16磅 |

### 智能公文导出

- 导出 A4 页面，上下左右边距严格按公文版心标准设置
- 支持段落、标题、无序/有序列表、表格、粗体、斜体的完整转换
- **两种标题编号模式**：
  - 保留原有编号（适合 AI 生成的、已含序号的内容）
  - 程序自动生成标准编号（自动为 h2 加"一、"，h3 加"（一）"，h4 加"1."）

### 完善的文件管理

- 打开 / 保存 / 另存为，支持 `.md` 和 `.txt` 格式
- **脏状态检测**：有未保存修改时，标题栏和按钮显示 ` ·` 提示
- **关闭前保存确认**：点击关闭按钮时，若有未保存内容，弹出确认对话框
- **双击文件直接打开**：在文件管理器中双击 `.md` 文件，软件自动启动并加载

### 辅助功能

- **文档大纲**：自动提取所有标题，显示于左侧侧边栏，点击快速跳转
- **实时字数统计**：分别统计中文字符数和英文单词数，显示于底部状态栏
- **键盘快捷键**：`Ctrl+S` 保存、`Ctrl+O` 打开、`Ctrl+E` 导出，主要操作不用鼠标

## 快速开始

### 方式一：下载安装包（推荐普通用户）

前往 [Releases 页面](https://github.com/yourname/tianma-md-editor/releases) 下载最新的 `.exe` 安装包，双击安装即可使用，无需配置任何开发环境。

### 方式二：从源码运行（开发者）

确保已安装 **Node.js 18+**，然后在 PowerShell 中执行：

```powershell
# 1. 设置国内镜像（必须最先做）
npm config set registry https://registry.npmmirror.com
$env:ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"

# 2. 克隆并安装
git clone https://github.com/yourname/tianma-md-editor.git
cd tianma-md-editor
npm install

# 3. 启动
npm run electron:dev
```

详细的开发环境搭建步骤请参考项目根目录的《开发者维护与操作手册》。

## 使用指南

### 推荐工作流：AI 生成 → 粘贴编辑 → 一键导出

1. 在 AI 对话框中生成公文内容，复制全部 Markdown 文本
2. 打开天马编辑器，清空默认内容，粘贴进来
3. 切换到「分栏」模式，对照右侧预览检查排版效果
4. 如需调整，直接在左侧编辑源码
5. 点击右上角「导出公文 Word」，选择标题编号方式，确认导出
6. 打开生成的 `.docx` 文件，格式已完全就绪

### 新建文档

启动软件后，编辑区自动载入一个示例公文模板，可直接清空开始写作。

### 打开文件

点击顶栏「打开」按钮（或 `Ctrl+O`），支持 `.md` 和 `.txt` 格式。也可以直接在文件管理器中双击 `.md` 文件关联打开。

### 保存文件

- **保存**：`Ctrl+S`，文件已有路径时直接覆盖；新建文档则弹出另存为对话框
- **另存为**：`Ctrl+Shift+S`

### 切换编辑模式

点击顶栏中间的三个按钮，或参考如下建议：

| 场景 | 推荐模式 |
|---|---|
| 日常写作、输入内容 | 所见即所得 |
| 检查最终排版效果 | 分栏预览 |
| 编写表格、代码或精确控制格式 | 源码 |

### 使用大纲导航

文档中包含标题时，左侧大纲面板自动出现。点击任意条目，编辑器立即跳转到对应位置。

## 公文导出说明

点击「导出公文 Word」（或 `Ctrl+E`），弹出导出对话框。

### 标题编号方式

**保留原有编号**：适合内容中已手动写好序号的情况，例如 AI 已生成了"一、（一）、1."格式，选此项后原样保留。

**程序自动生成**：适合标题没有编号，或 AI 生成的编号有误、混乱的情况。软件会先剥除原有编号，再按层级重新生成标准格式：

```
## 工作总结        →  一、工作总结
### 主要成绩        →  （一）主要成绩
#### 经济指标       →  1. 经济指标
```

### 导出文件的版面规格

| 项目 | 规格 |
|---|---|
| 纸张 | A4（210mm × 297mm） |
| 上边距 | 37mm |
| 下边距 | 35mm |
| 左边距 | 28mm |
| 右边距 | 26mm |
| 正文行距 | 固定值 28磅 |

## Markdown 语法参考

### 标题

```markdown
# 文件标题（一级，方正小标宋，居中）
## 第一部分（二级，黑体）
### 第一节（三级，楷体）
#### 细分项（四级，仿宋加粗）
```

### 正文段落

```markdown
直接输入文字即为正文。预览和导出时自动使用仿宋字体，首行缩进 2 字符。

空一行开始新段落。
```

### 强调

```markdown
**加粗文字**
*斜体文字*
```

### 列表

```markdown
- 第一条
- 第二条
    - 子项（4个空格缩进）

1. 有序第一项
2. 有序第二项
```

### 表格

```markdown
| 项目 | 内容 | 备注 |
|------|------|------|
| 单位 | XX局 | — |
| 日期 | 2025年 | — |
```

### 引用块

```markdown
> 这是一段引用内容。
```

### 分割线

```markdown
---
```

## 项目结构

```
tianma-md-editor/
├── main.cjs              # Electron 主进程（窗口、IPC、文件读写）
├── preload.cjs           # 预加载脚本（contextBridge 桥接）

├── package.json          # 依赖声明、npm 脚本、打包配置
├── index.html            # 应用 HTML 入口
├── src/
│   ├── App.jsx           # 主组件（全部 UI 逻辑、编辑器、导出）
│   └── index.css         # 全局样式（界面框架 + .prose 公文排版）
├── public/               # 图标等静态资源
├── dist/                 # Vite 构建产物（自动生成，不提交 Git）
└── dist_electron/        # electron-builder 打包产物（不提交 Git）
```
